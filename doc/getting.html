Getting Chromium

Chromium is maintained at SourceForge.  In addition to the source code, you'll also find a bug tracking system and three mailing lists.  Even if you're not a developer, you may wish to subscribe to chromium-announce to receive information about updated releases.

Obtaining source code

To get the source code, you do not have to be a registered SourceForge user.  You can get the source in one of two ways:

First, you can download the actual alpha release snapshot, which is available from the main project page, under "Latest File Releases".  This will be a gzip'ed tar file of the entire source tree on the release date, along with a copy of this documentation.  Simply extract the archive and you've got everything.

Alternately, you can check out the top-of-tree source code from the CVS repository, if you want to live on the bleeding edge.  Between the time of this release and the BETA release scheduled for October, many bugs will surely be fixed, and many new features and modules will be added.  Announcements will be sent to chromium-announce when particularly interesting features are available or particularly bad bugs are fixed.  To check out the source code, just follow the instructions on the CVS repository page.  If you're unfamiliar with CVS, useful documentation is available in Open Source Development with CVS.

Compiling the source

All of Chromium lives under a directory called "cr".  Change to that directory, and type "make".  You will need Python 2.0 or later, and Perl 5.0 or later to build on any platform.  Binaries are put in cr/bin/<ARCH>, where <ARCH> is the name of your architecture (Actually, it's what's returned by the "uname" command.  Except on Windows, where cygwin's "uname" returns something long and silly, so it gets re-written to "WIN_NT".).  You should add this directory to your PATH.

NOTE: If you do not have Myrinet GM libraries, you should turn off GM support in the file cr/util/Makefile before building.  There are instructions in that file.

Platform specific notes:

  • Any UNIX: Shared libraries will be put in the directory cr/lib/<ARCH>. You will need to add this directory to your LD_LIBRARY_PATH environment variable.  Chromium builds a lot of shared libraries, so you can't skip this step.
  • Windows:  You will also need to install the cygwin tools to build Chromium.  If you're building from the Windows shell (as opposed to the cygwin bash implementation), you will need to set the environment variable "MAKE_MODE" to the string "unix".  This lets gmake use bash as its subshell instead of the Windows shell, which is required.  Shared libraries on Windows get put in cr/bin/WIN_NT, which (since it's on your path, right?) will be searched automatically, so there's no LD_LIBRARY_PATH silliness.

Note that Chromium version ALPHA does not support byte-swapping of arguments, so all machines involved in the cluster have to have the same endianness.  This will be fixed for the next release.
