Getting Help

Chromium ALPHA is essentially a technology preview.  Many of the features work, but some of them don't.  See the "List of Known Bugs" page for a list of things that I know about today.

If you encounter something that you think isn't working right, and it's not on the bug list, you should file a bug report on the SourceForge project bug tracking page.  Please browse the existing bugs first to make sure you're not submitting a duplicate.

If you don't have a bug, but Chromium doesn't do something you think it ought to, you can submit a feature request on the SourceForge project feature request page.  You can check out the "Things To Do For Version BETA" page to see if it's already on that list.  Feature requests will (obviously) be lower priority than bugs, but they'll all be taken seriously.  Of course, if you have a feature request, you could give thought to implementing it yourself if you have the resources.  This is an open-source project, after all.  Feature additions may be rolled into the main source tree on a case-by-case basis.

If you need to contact me directly, just send me mail at humper@graphics.stanford.edu.  Remember, I'm only one man, so please use this wisely.

There are also three mailing lists, available from the SourceForge project mailing list page.  chromium-announce is for announcements relating to this project.  It's very low-traffic, and everyone should subscribe to this mailing list for critical update notification, BETA release announcements, etc.  chromium-users is for users of the system who are not necessarily hacking on the code.  Finally, chromium-dev is a mailing list for people actively developing the system.
